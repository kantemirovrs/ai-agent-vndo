**4. Проблема:**
Проект направлен на решение проблемы оперативного поиска релевантной информации по БД Внутренних нормативных документов (ВНД), с учётом их исторической актуальности. На текущий момент времени, все сотрудники компании, использующие в своей профессиональной деятельности положения ВНД, для поиска необходимой информации, должны знать реквизиты целевого документа, место его расположения. При этом, релевантная информация может содержаться в нескольких документах, что требует от пользователя знания всей номенклатуры ВНД. Таким образом, отсутствие семантического подбора информации, в значительной степени усложняет процесс поиска, извлечения, анализа информации, может стать причиной всевозможных пользовательских ошибок и, как следствие этого, материального либо репутационного ущерба.

**Масштаб проблемы:**
Примерно  60% персонала работают с ВНД на постоянной основе, остальные 40% сотрудников взаимодействуют периодически (ознакомление с приказами, инструкциями, распоряжениями и т.д.). В силу большого объёма нормативной документации, ознакомление с содержанием самих ВНД носит формальный характер. Кроме того, держать в памяти все детали огромного разнообразия ВНД не представляется возможным. При этом, практически все аспекты производственной деятельности, строго регламентированы именно в этих нормативных документах, их нарушение (по незнанию, халатности) может привести к административным, дисциплинарным взысканиям, материальному или репутационному ущербу.         

**5. Решение:**
Создание корпоративного ресурса, на котором пользователь смог бы, самостоятельно сформулировав свой запрос к ИИ-агенту, получить достоверную информацию из ВНД (приказов, распоряжений, инструкций),  с указанием самих ВНД, которые легли в основу данного ответа. При этом, должна быть реализована возможность исторической актуализации ВНД, в соответствии со спецификой запроса пользователя (к какому временному периоду обращён запрос).

**Что делает LLM в вашем решении?**
Проект предполагает использование LLM, технологий RAG, векторной БД. В связи с со спецификой проекта, его задач, объективных и субъективных составляющих, использование LLM представляется наиболее оптимальным решением данной задачи. Основная задача LLM – это генерация ответа на пользовательский запрос, построенный исключительно на контексте ВНД.     

**Технический подход**
При реализации проекта, предполагается использовать модель с открытым исходным кодом Qwen3-30B-A3B-Instruct-2507. Так как она предлагает оптимальный баланс эффективности и возможностей с большим контекстным окном, что делает ее идеальной для приложений RAG, которым необходимо обрабатывать огромные коллекции документов, сохраняя при этом экономическую эффективность.
Для хранения векторных представлений ВНД планируется использовать векторную БД Qdrant.

**Что будет, если не делать проект?**
С учётом ежегодного роста ВНД, высокого кадрового оборота, требованиями современного делопроизводства, потребность в ИИ-агентах, обеспечивающих быстрый и качественный, семантический поиск на всём многообразии ВНД, будет только возрастать. Без реализации данного проекта, поиск необходимой информации будет по-прежнему отнимать у сотрудников много времени, а в силу человеческого фактора, высока вероятность использования не релевантных данных, которые, в свою очередь могут привести к существенным материальным либо репутационным издержкам.

Главные риски:

Решение: выделение дополнительных ресурсов инфраструктуры, выбор более легковесных моделей.

Решение: дополнительный мониторинг сетевой активности, использование контейнеризации, применение политик безопасности AD/GPO.
   
3) Векторизация большого числа ВНД. 
Решение: разделение ВНД по тематическим кластерам, поэтапная векторизация.
